# fig CLI — MVP定義

## コンセプト

ターミナルに**インテリジェントな自動補完（autocomplete）**を追加するOSS CLIツール。
[Fig（YC S20、2023年AWS買収）](https://www.ycombinator.com/companies/fig)にインスパイアされたRust製の実装。

## 解決する課題

1. **アクセシビリティ**: 初心者にとってターミナルは敷居が高い。補完があれば使いやすくなる
2. **生産性**: 経験者でもサブコマンドやフラグを全て覚えるのは困難。補完で効率化

## MVPスコープ（v0.1）

### Must Have

| # | 機能 | 説明 |
|---|------|------|
| 1 | コマンド自動補完 | 既存CLIツール（git, docker, npm等）のサブコマンド・フラグを補完 |
| 2 | 補完スペック | TOML形式で補完定義を記述できる仕組み |
| 3 | シェル統合 | zsh / bash にフックして補完をインジェクト |
| 4 | ビルトインスペック | git, docker, npm など主要CLIの補完スペックを同梱 |

### Out of Scope（v0.1では対象外）

- GUI/TUI リッチUI
- App Store / プラグインエコシステム
- リモートからのスペック配信
- fish / nushell 対応
- AIベースの補完

## 技術スタック

| 項目 | 選定 | 理由 |
|------|------|------|
| 言語 | Rust | パフォーマンス、安全性、シングルバイナリ配布 |
| CLIフレームワーク | `clap` | Rustエコシステムのデファクト |
| 補完スペック形式 | TOML | 人間が読み書きしやすい |
| シェル統合 | zsh widget / bash readline | 主要シェルをカバー |
| 配布 | `cargo install` / Homebrew | Rust開発者 + macOSユーザーをカバー |

## マイルストーン

### M1: プロジェクト初期化

- Cargoプロジェクト作成
- CLIスケルトン（`fig init`, `fig --version`）
- CI/CD セットアップ（GitHub Actions）

### M2: 補完スペックパーサー

- TOML形式の補完スペック定義を設計
- パーサー実装（TOML → 内部データ構造）
- ユニットテスト

### M3: シェル統合

- zshにフック（Tab押下で補完候補を表示）
- bashにフック
- `fig init` でシェル設定を自動追加

### M4: ビルトインスペック

- git の補完スペック作成
- docker の補完スペック作成
- npm の補完スペック作成

### M5: インストール/アンインストール

- `fig install` でシェル設定の自動追加
- `fig uninstall` でシェル設定の自動削除
- 初回セットアップのガイド表示
